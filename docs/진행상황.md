### 3-2. 공통·레이아웃

- ✅ 디자인 시스템: 포레스트 그린(#306D29)/크림(#FBF5DD) 라이트 + 딥 차콜 그린 다크모드 (시스템 설정 자동 감지)
- ✅ 폰트: Syne ExtraBold(로고) + Noto Sans KR (Google Fonts, index.html에서 로드)
- ✅ 하단 탭바: [홈][탐색][+FAB][알림][마이], 터치 영역 44px 이상
- ✅ PC(1200px+) 레이아웃: 그린 그라데이션 배경 + 좌측 브랜드 소개 패널 + 480px 폰 프레임(그림자) — 하단 탭바와 프레임 중앙 정렬 유지
- ✅ 라우팅: HashRouter (GitHub Pages 새로고침 404 방지)

### 3-3. 설계 변경 사항 (DB 문서와 다른 점)

- **인증**: `petlog_users.password` 직접 저장 대신 **Supabase Auth** 사용.
  `petlog_users`는 `auth.users(id)`와 1:1 연결된 프로필 테이블 (email, nickname, birth_date).
  가입 시 nickname/birth_date를 auth 메타데이터에 저장 → 첫 로그인 시 프로필 행 자동 생성(`use-auth.jsx`의 ensureProfile).
- **이미지**: 업로드 대신 Unsplash API 검색으로 URL 저장 (DB 문서에 명시된 실습 대체 방식).

`petlog_hashtags`, `petlog_post_hashtags`

- 트리거: 좋아요/댓글 수 캐시 자동 갱신, 게시물 updated_at 갱신, 답글 1단계 제한

- 마이그레이션 이력: `petlog_mvp_schema` → `petlog_triggers_and_rls` → `petlog_security_hardening`

## 6. 코드 구조 참고

```
petlog_sns/src/
├── lib/supabase.js              # Supabase 클라이언트
├── hooks/use-auth.jsx           # 세션·보호자 프로필·펫 목록·활성 펫 전역 상태 (AuthProvider)
├── utils/
│   ├── posts-api.js             # 게시물·좋아요·팔로우·해시태그 API
│   ├── comments-api.js          # 댓글·답글·이모지 반응 API
│   ├── unsplash-api.js          # Unsplash 사진 검색
│   └── format-date.js           # 상대시간·나이·생일 D-day
├── components/
│   ├── common/                  # app-header, bottom-nav, main-layout, pet-avatar, protected-route
│   └── ui/                      # post-card, image-slider, comment-item, pet-form, unsplash-picker, empty-state
├── pages/                       # login, signup, onboarding, home, explore, post-detail,
│                                #   create-post(작성/수정), profile(마이+타펫), pet-form(펫 추가/편집), notifications
├── theme.js                     # 라이트/다크 테마 팩토리 getTheme(mode)
└── App.jsx                      # 라우팅 + PC 브랜드 패널 + 480px 프레임
```

- 활동 주체: 좋아요·댓글·팔로우는 항상 **현재 선택된 펫(activePet)** 기준 (`useAuth()`)
- 좋아요/댓글 수는 클라이언트에서 직접 갱신하지 않음 — DB 트리거가 캐시 유지
- 새 페이지 추가 시: `ProtectedRoute`로 감싸고, 탭바 노출 페이지는 `MainLayout` 라우트 아래에 배치
